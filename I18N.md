# Internationalization (i18n) Guide

This app ships **Traditional Chinese (Taiwan) as the default language**, with an
English fallback the user can toggle. **All user-facing text must go through the
translation system — never hardcode a display string.**

If you add a screen, button, label, placeholder, toast, or empty-state and you
typed the English (or Chinese) words directly into the JSX, that's a bug: it
won't translate and it breaks the language toggle.

---

## TL;DR — adding any new text

1. Add the key to **both** `en` and `zh` in `src/lib/translations.ts`.
2. In the component: `const { t } = useLang()` then render `{t('your.key')}`.
3. `npx tsc --noEmit` — TypeScript fails the build if a key is missing in `zh`
   or if you `t('typo')` a key that doesn't exist.

```tsx
'use client'
import { useLang } from '@/contexts/LanguageContext'

export function MyThing() {
  const { t } = useLang()
  return <button>{t('common.join')}</button>   // ✅
  // return <button>Join</button>              // ❌ never do this
}
```

## How the system works

- **`src/lib/translations.ts`** — the single source of truth. Two objects, `en`
  and `zh`. `const zh: typeof en = { ... }` means **TypeScript forces `zh` to have
  the exact same keys as `en`.** A missing or extra key is a compile error.
  `TranslationKey` is the union of all valid keys.

- **`src/contexts/LanguageContext.tsx`** — provides `useLang()`:
  - `t(key)` → the string for the current language (falls back to the key name).
  - `lang` → `'zh' | 'en'` (default `'zh'`).
  - `toggleLang()` → flips language, persists to `localStorage` (`caddie_lang`).
  - Default is `'zh'`; the saved choice is adopted after mount (hydration-safe).

- **`<html lang="zh-Hant">`** is set in `src/app/layout.tsx`.

## Patterns (copy these)

### Plain text, attributes, labels
```tsx
<span>{t('home.findRound')}</span>
<input placeholder={t('chat.searchPlaceholder')} />
<div title={t('common.profile')} />
<img alt="" />            {/* decorative images: empty alt, not English */}
```

### Interpolation / counts
Build the string around `t()` — don't bake values into the dictionary:
```tsx
{`${openSpots} ${t('home.spotsOpen')}`}     // "3 個空位"
{count} {t('community.members')}            // "12 位成員"
```

### Plurals
**Chinese has no singular/plural distinction.** Don't add `xxxOne` / `xxxMany`
keys. Use one "suffix" key for both:
```tsx
{feed.length} {t('community.detail.postsSuffix')}   // 1 → "1 則貼文", 5 → "5 則貼文"
```
(English will read slightly oddly for "1 Posts" — that's an accepted trade-off
since Chinese is the primary language.)

### Module-scope constant arrays / maps
A `const X = [...]` defined **outside** the component can't call `t()`. Store the
**translation key** in the array and translate at the render site:
```tsx
import type { TranslationKey } from '@/lib/translations'

const POST_TYPES: Array<{ key: PostType; labelKey: TranslationKey }> = [
  { key: 'round_report', labelKey: 'post.type.roundReport' },
  { key: 'seeking',      labelKey: 'post.type.seeking' },
]

// inside the component:
{POST_TYPES.map(p => <span key={p.key}>{t(p.labelKey)}</span>)}
```
For a lookup map keyed by a runtime value, guard the lookup:
```tsx
const TYPE_LABEL_KEYS: Record<string, TranslationKey> = { round_report: 'post.type.roundReport', /* … */ }
{t(TYPE_LABEL_KEYS[post.type] ?? 'post.type.general')}
```

### Error toasts that show backend messages
The backend already returns **Traditional Chinese** error strings (see below).
Keep the localized fallback for the network-failure case:
```tsx
setError(e instanceof Error ? e.message : t('error.saveFailed'))
```

### Dates, times, money, golf formats — use the helpers, don't reformat inline
`src/lib/utils.ts` is **language-aware** (it reads a module-level language kept in
sync by `LanguageProvider` via `setFormatLang`). Always use these instead of
`toLocaleDateString('en-US', …)`:

| Helper | Output |
|---|---|
| `formatDate(iso)` / `formatTeeTime(iso)` | localized date / time |
| `timeAgo(iso)` | "3 小時前" / "3h ago" |
| `formatMoney(cents)` | "NT$2,400" / "免費" |
| `formatFormat(fmt)` / `formatHcpReq(req)` | pulls from the `format.*` / `hcp.*` keys, so it always matches the pickers |

When you must call `toLocaleDateString` directly (e.g. a custom calendar header),
branch on `lang`:
```tsx
const { t, lang } = useLang()
d.toLocaleDateString(lang === 'zh' ? 'zh-TW' : 'en-US', { month: 'long', year: 'numeric' })
```

## Key naming conventions

Namespaced, dot-separated, `lowerCamel` leaf:

```
nav.*            bottom nav
auth.*           auth flow   (auth.error.* for auth error toasts)
home.* rounds.* community.* chat.* profile.* host.* manage.*   per-screen
round.*          round-detail bits      post.* / post.detail.*  posts
community.type.* community.privacy.* community.detail.* community.create.*
sheet.account.* sheet.hcp.* sheet.compose.* sheet.notif.*       bottom sheets
format.* hcp.* holes.* day.* month.* monthFull.*                enumerations
loading.* error.* success.*                                     status text
common.*         genuinely shared words (join, full, back, cancel, host, player, you, holesSuffix…)
ui.*             low-level UI (avatar alt, date picker)
```

Reuse an existing key before inventing a new one (e.g. `community.view`,
`common.full`, `loading`). Suffix keys for "{n} <word>" patterns end in `Suffix`.

## Backend (`../backend`)

The backend has **no `t()`** — it serves only this Chinese app, so user-facing
strings are written directly in Traditional Chinese:

- Error responses (`c.json({ error: '找不到球局' }, 404)`) — the frontend shows
  `error.message` straight from these.
- Notification `title` / `body` (in `src/routes/*.ts`) — rendered in the
  notifications panel.

Keep interpolated variables (names, course names) intact; translate only the
human-readable words. Leave `console.*` logs, enum values, and field keys English.

## Verifying your change

```bash
cd frontend
npx tsc --noEmit          # fails on missing/extra zh keys and on t('badKey')
npm run build             # full build incl. lint + static generation
```

Quick grep for accidental hardcoded text (should return nothing meaningful):
```bash
grep -rnoE '>[ ]*[A-Z][a-zA-Z]+[^<]*<' src --include='*.tsx' \
  | grep -vE '<(svg|path|circle|rect|line|polyline|polygon)|\{t\(|Promise<'
```

## Translation tone

Traditional Chinese (Taiwan, zh-Hant). Match the existing voice: concise,
friendly, golf-appropriate (球局 = round, 差點 = handicap, 果嶺費 = green fee,
比桿賽 = stroke play, 開球時間 = tee time). Use Taiwan terms and full-width
punctuation in sentences (，。！？（）).
